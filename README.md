# Кибериммунный светофор (учебный пример)

Учебный проект по кибериммунному подходу: взаимодействие компонентов светофора проходит через монитор безопасности с политикой **default deny**.

Проект предназначен для обучения: часть тестов специально красные, пока студент не выполнит задания.

## Авторство и источник идеи

- Оригинальная идея учебного примера: **Сергей Соболев**.
- Текущая адаптация под формат проекта (Python + pytest): **Юрий Шишкин**.
- Основа: https://github.com/cyberimmunity-edu/cyberimmune-systems-example-traffic-light-jupyter-notebook

## Что в репозитории

- `main.py` — основной учебный сценарий (консольная демонстрация).
- `tests/test_ipc_policies.py` — базовые тесты IPC-политик.
- `tests/test_mode_checker.py` — базовые тесты проверки режимов.
- `tests/test_student_tasks.py` — тесты по заданиям 1–5.

## Запуск

```bash
python main.py
```

## Тесты

Установите `pytest`, затем:

```bash
pytest -q
```

Раздельный запуск:

```bash
pytest -q tests/test_ipc_policies.py tests/test_mode_checker.py
pytest -q tests/test_student_tasks.py
```

Примечание:
- базовые тесты должны быть зелёными;
- часть тестов `test_student_tasks.py` специально падает до выполнения заданий.

## Учебная логика

- Монитор принимает события от сущностей и проверяет политики.
- Разрешены только явно описанные маршруты и команды.
- Неразрешённые сообщения блокируются по принципу **default deny**.
